## Tasks (in dependency order, each 5-15 min of focused work)

### Task 1 — MarketContext object: build the "human dashboard" the strategies see

**Create:** `market_context.py`

A pure data class assembled per market per loop. Bundles every variable a human trader would have on the screen.

Fields:
- `market_id`, `slot_id`, `asset`, `interval_minutes`, `outcome_labels`
- `now_ts`, `end_ts`, `seconds_to_expiry`, `slot_age_seconds`, `tte_pct` (1.0 → 0.0)
- `tte_bucket` ("early" >180s, "mid" 60-180s, "late" 30-60s, "endgame" <30s)
- `mid_price_yes`, `mid_price_no`, `best_bid_yes`, `best_ask_yes`, `book_spread_bps`, `top_depth_yes`, `top_depth_no`, `imbalance_yes`
- `last_trade_price`
- `spot_price`, `spot_move_pct_window` (asset spot move since slot start, signed)
- `momentum_score` (smoothed slope of mid in last N samples, signed)
- `recent_mid_history` (deque of last 30 mids for diagnostics)

Builder function:
```python
def build_market_context(market: dict, orderbook: OrderBook, now_ts: float,
                         spot_provider, mid_history_store) -> MarketContext: ...
```

`spot_provider` is an injected callable returning latest spot price for asset (Binance public REST, cached per second).
`mid_history_store` is a small in-process dict keyed by `(market_id,outcome)` storing recent mids.

**Also add:** `spot_provider.py` — a tiny spot price client.
- Polls Binance public REST `/api/v3/ticker/price?symbol=BTCUSDT` for BTC, ETH, SOL, XRP every 2 seconds.
- Caches and returns the most recent value.
- Pure stdlib + requests, no async needed (called from sync path; runs a background thread).
- Records spot price at slot_start for each (asset, slot_id) so spot_move_pct_window is well-defined.

**Tests:**
- `tests/test_market_context.py`:
  - context fields populated correctly from a sample market+orderbook
  - tte_bucket transitions correctly across thresholds
  - momentum_score is positive when mid trends up, negative when down
  - spot_provider falls back to last known value on network error

**Wire-in:** No strategy uses it yet; this task just lands the layer.

### Task 2 — TTE-aware quoting in toxicity_mm

**Modify:** `strategies/toxicity_mm.py` and `cli.py` (call site).

Change `generate_quotes` signature to accept an optional `MarketContext` and use it when present (backward-compatible default = None).

Behavior changes when context is present:

1. **TTE-scaled spread.** At TTE > 180s: base spread. At TTE 60-180s: spread × 1.3. At TTE 30-60s: spread × 1.8. At TTE < 30s: spread × 3.0 OR refuse to quote.
2. **TTE-scaled size.** size *= (0.4 + 0.6 * tte_pct). Cuts late-window inventory build-up.
3. **Endgame flatten.** At TTE < 30s: do NOT add new resting orders. If position is non-zero, emit a flatten-quote at slightly aggressive cross price (best_bid + 1 tick to sell, best_ask - 1 tick to buy).
4. **Inventory skew.** If existing inventory on this outcome > 0, lower the bid price and raise the ask price (push inventory back toward flat). Use existing `inventory_adjust_price` helper.
5. **Book-imbalance fade.** If imbalance_yes > 0.6, lean ASK heavier, BID lighter; vice versa for negative imbalance.

CLI call site: build a `MarketContext` per market per loop and pass into `mm.generate_quotes(..., context=ctx)`.

**Tests:**
- `tests/test_toxicity_mm_tte.py`:
  - TTE > 180 and TTE = 60 produce different spreads
  - TTE < 30 produces no new quote (or flatten quote only)
  - Inventory skew biases the right side
  - Backward compat: no context → exact legacy behavior

### Task 4 — Spot-momentum directional bias

**Create:** `strategies/spot_momentum.py`

Thesis: a 5m crypto-up/down market is a derivative of spot. If BTC spot just moved +0.4% in the last 90s with 120s left in the window, "Up" is mispriced if it's below ~0.65.

Logic:
1. Activated only on 5m markets (interval_minutes == 5).
2. Needs `MarketContext.spot_move_pct_window` ≥ +0.20% AND tte_pct in [0.2, 0.7] (i.e. mid-window) → BUY "Up" if mid_yes < 0.70.
3. Negative mirror for Down.
4. Variable size: notional = 2.0 + 4.0 * |spot_move_pct| (capped at 6.0).
5. One-fire-per-slot.
6. Always-respect existing exposure + governance gate.

Add to candidates first; manual flip to active after smoke test.

**Tests:**
- `tests/test_spot_momentum.py`:
  - +0.3% spot move, 200s left, mid=0.6 → BUY Up at sized notional
  - +0.3% spot move, mid=0.85 → no trade (already priced in)
  - -0.3% move, mid=0.4 → BUY Down
  - 15m market → no signal
